## 2. 场景理解 —— 最核心的gap

### 2.1 设计要求

论文将三种遗忘类型嵌入到真实医疗场景中：

| 遗忘类型 | 医疗含义 | 核心难点 |
|---|---|---|
| Node Unlearning | 患者撤回授权、机构退出多中心建模、疾病节点删除 | 删除节点→邻居断裂→MIA可检测疤痕 |
| Edge Unlearning | 医学证据更新（旧药物-疾病关系被证伪）、PPI边撤销 | 删除边→度分布偏移→结构疤痕 |
| Feature Unlearning | 敏感基因特征、人口学变量不可使用 | 特征删除→锚定目标必须选h_new |

### 2.2 学生现状

- **技术层面**：三种遗忘类型的代码全部打通，270主实验覆盖了node/edge/feature
- **理解层面**：学生对这些操作在医疗场景下的真实含义还没有深入把握

### 2.3 诊断

学生在以下三个关键认识上存在gap：

| 学生理解 | 实际要求 |
|---|---|
| "删一个节点" | "删一个患者/疾病/药物节点后，如何避免破坏医学知识骨架" |
| "删一条边" | "删除一个被最新证据证伪的药物-疾病关系后，如何不留下结构异常" |
| "删一个特征维度" | "删除基因检测指标后，如何防止模型通过邻居聚合'恢复'这个特征" |

**关键诊断**：学生把Unlearning理解为"通用机器学习任务"（删节点/边/特征），但**没有理解这些操作在医疗场景下的隐私动机和结构后果**。这导致实验设计是benchmark导向而非问题导向。

## 3. 任务设计 —— 缺乏RQ驱动的系统性规划

### 3.1 设计要求（6个RQ）

| RQ | 定位 | 核心问题 | 当前状态 |
|---|---|---|---|
| RQ1 | P0 | Hub deletion是否比random deletion更难？ | 进行中18/36 |
| RQ2 | P0 | HASI在主对比中是否综合最优？ | 已完成但口径偏差 |
| RQ3 | P1 | Anchor Stabilization消融 | 未开始 |
| RQ4 | P1 | Structural Inpainting消融 | 未开始 |
| RQ5 | P1 | DAR stress test | 未开始 |
| RQ6 | P2 | 多类型遗忘scope验证 | 主实验已覆盖 |

### 3.2 诊断

学生当前的实验推进顺序存在逻辑问题：

- **已完成**：RQ2（主对比）+ RQ6（scope）——但这是"证明方法好"的部分
- **缺失**：RQ1（证明问题存在）——这是"为什么需要这个方法"的部分
- **完全缺失**：RQ3/4/5（证明各组件的必要性）——这是"方法为什么work"的部分

正确的逻辑链路应该是：
```
RQ1（问题存在） → RQ2（方法好） → RQ3/4（组件必要性） → RQ5（核心机制）
```

但学生跳过了RQ1和RQ3-5，直接完成了RQ2。这会导致：
- Introduction中"Hub damage更严重"的claim缺少实验证据
- 方法部分缺少消融支撑
- DAR核心贡献无法验证

## 4. 指标体系 —— 框架确立但细粒度缺失

### 4.1 设计要求

**硬性要求：不能只报accuracy。** 主表必须同时报告 Utility + Structural + Privacy + Efficiency 四个维度。

### 4.2 学生现状

进展文档第8.2节列出了四维指标框架：
- Utility: accuracy_drop ✅
- Structure: degree_kl_divergence, cc_change, component_count_change ✅
- Privacy: overall_mia_auc, privacy_score ✅
- Efficiency: unlearn_time_seconds ✅

### 4.3 诊断

| 问题 | 评语 |
|---|---|
| 四维框架意识 | 已建立，进展文档明确列出了所需指标 |
| 是否已纳入主实验聚合 | **无法从进展文档确认** aggregate_summary.csv中是否实际包含这些列 |
| Baseline MIA是否全覆盖 | 进展文档暗示"retrain和GraphEraser已有MIA对比"(§8.3)，但不确定是否270个baseline实验全部有MIA结果 |
| 细粒度指标缺失 | Hub embedding drift, neighbor drift, 三级MIA(weak/medium/strong) 未在进展文档中提及 |

**关键诊断**：学生知道需要报什么指标，但可能存在"知道归知道，实际聚合csv里只有accuracy"的情况。**baseline MIA是否100%补齐是P0级别的核查事项。**

## 5. 医疗数据集缺失 —— 最明显的gap

### 5.1 设计要求

三层医疗数据集架构：

| 层级 | 数据集 | 要求 | 实际 |
|---|---|---|---|
| Layer 1 | PubMed | 最低成本医疗相关图 | ✅ 已接入 |
| Layer 2 | PPI / ogbn-proteins | Hub有生物学意义 | ❌ 零 |
| Layer 3 | PrimeKG / TDC-style KG | 最强医疗叙事 | ❌ 零 |

**最小可投稿版本：PubMed + PPI 双数据集。**

### 5.2 诊断

| 当前数据集 | 类型 | 医疗语义 | 能否支撑医疗叙事 |
|---|---|---|---|
| Cora | 学术引用网 | 无 | ❌ |
| CiteSeer | 学术引用网 | 无 | ❌ |
| PubMed | 生物医学文献引用 | 弱 | 仅第一层，不能单独支撑 |

目前270个主实验中，只有1/3（PubMed部分）具有医疗关联。Cora/CiteSeer完全是非医疗数据。

**如果学生以为"PubMed=医疗数据集就够了"，这是对论文故事线的严重误解。** PubMed是biomedical literature graph，不是临床医疗图。它适合RQ1/RQ2的初步验证，但不能单独支撑"医疗图遗忘"的叙事。

**PPI或ogbn-proteins的接入是硬性要求**，否则：
- "医疗Hub有生物学意义"的claim无法验证
- "结构感知的外科式编辑"在生物网络上的泛化性无法证明
- Introduction中丰富的医疗场景（蛋白互作、基因通路、药物靶点）将缺乏实验支撑

### 6.2 关键gap详析

#### Gap 1: 主实验口径偏差（E2）

| | 设计故事线要求 | 实际完成 |
|---|---|---|
| Node删除来源 | 全图**random_all** paper节点删除 | **random_train**（训练标签节点） |
| PubMed遗忘节点数 | 应为较大的全图比例（如k=60/120） | 仅3/6个节点（因train_mask仅60节点） |
| 论文表述 | "random PubMed paper deletion over the full graph" | 只能写"random training-node deletion under Planetoid splits" |

这个gap不导致实验结果无效，但会**削弱医疗故事线的说服力**：删3个PubMed训练节点对模型的冲击远小于删60个全图Hub节点。

#### Gap 2: DAR核心贡献悬空（E5）

DAR是论文第三个核心贡献：「当Primary Hub必须删除时，DAR比单点替代更隐私安全」。但当前：
- DAR仅在随机删除实验中被动触发（触发率极低）
- 没有构造Primary-heavy forget set（top-k HubScore节点）
- 没有比较单点替代vs分散替代的六种策略
- 没有Pareto curve（utility drop vs MIA AUC）

**无E5则论文第三个贡献不成立。**

#### Gap 3: 消融实验缺失（E3/E4）

- 无Anchor消融 → 无法证明分层锚定的必要性
- 无ERF消融 → 无法证明PPR定位的优势
- 无GSI消融 → 无法证明结构修复降低MIA风险

**无消融则方法部分的每个组件都无法独立论证价值。**

#### Gap 4: 医疗数据集验证（E6）

无PPI/ogbn-proteins → "医疗图遗忘"核心叙事空悬在PubMed上（PubMed严格来说是文献引用图，不是医疗图）。

## 8. 给学生的三个核心认知纠正

### 8.1 Unlearning不是"删掉就完事"

**错误理解**：Unlearning = 从训练集中移除节点/边/特征 + 微调模型 → 看accuracy恢复多少

**正确理解**：Unlearning = **在不留下MIA可检测痕迹的前提下**移除信息 → 需要同时验证 privacy（攻击者是否猜出来删了什么）和 structure（删除是否像外科手术而非暴力拆迁）

医疗场景额外要求：删除后**医学知识骨架**不被破坏（如删除一个糖尿病节点不影响血糖-药物-并发症的关联推理）。

### 8.2 实验不是为了跑满矩阵，而是为了回答RQ

**错误做法**：按数据集×类型×比例×种子×方法的笛卡尔积填满所有组合

**正确做法**：每个实验有明确要回答的RQ，实验设计围绕RQ展开：
- RQ1需要 matched-count 三类对比（不是ratio型随机删除）
- RQ5需要 Primary-heavy stress test（不是等DAR偶然触发）
- RQ3/4需要消融变体（不是只有HASI-full一个配置）

### 8.3 论文故事线必须先于实验设计

**错误顺序**：先跑完所有能跑的实验 → 再想故事线 → 发现数据和故事线不匹配

**正确顺序**：
1. 确定故事线（医疗图遗忘，已由导师完成）
2. 设计RQs（6个RQ）
3. 确定每个RQ的实验设计（数据集/删除方式/对比方法/指标）
4. 按P0→P1→P2优先级执行实验
5. 用实验结果反过来调整故事线的表述

学生当前的问题是第2、3步缺失，直接从第4步开始跑矩阵。
