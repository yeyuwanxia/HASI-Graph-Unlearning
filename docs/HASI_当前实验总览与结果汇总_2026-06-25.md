# HASI 当前实验总览与结果汇总（2026-06-25）

本文档由当前仓库本地 artifacts 重新汇总，主结果只使用当前 shared-base 评估层：`results/pubmed_eval`、`results/primekg-disease-gene-small-nosource_eval`；PubMed RQ 使用 `results/rq/pubmed`。历史结果目录只作为状态说明，不和当前主表混算。

## 1. 总结论

- 当前最适合论文主实验的数据集是 `pubmed` 和 `primekg-disease-gene-small-nosource`。两者都已按 shared-base 协议完成 node / edge / feature 三类遗忘任务，ratio 为 0.05 和 0.10，base seed 为 42 / 123 / 2024。
- 当前主协议是：shared_base 用 `stratified_random` 的 60/20/20 节点划分，训练阶段使用 `train_subgraph`；node forget 只从 train nodes 抽，edge forget 按当前协议从指定 edge scope 抽，feature forget 抽特征维度；所有方法复用同一个 forget set 和同一个 shared_base artifact。
- `primekg-disease-gene-small-nosource` 是当前医学 KG 主线。早期 `primekg-disease-gene-small` 带 `node_source_one_hot`，source 与标签高度相关，已归入 archived/legacy，不建议作为论文主表。
- PPI single-label 适配线不建议继续进入 HASI 主实验：most-frequent projection 只有 3 类且训练多数类约 92%；balanced20 虽然类别均衡，但 GCN/GraphSAGE train accuracy 约 5%，没有形成有效 utility；fullgraph GraphSAGE 虽能到约 48% test accuracy，但不符合 strict `train_subgraph` 主协议。
- `ogbn-arxiv` 已按你的决定从代码和 artifacts 中移除，不再纳入当前实验。Cora/CiteSeer 仍有 shared_base 和历史结果，但属于非医学/历史层，不作为当前医学主线证据。

## 2. 指标口径

- `accuracy_after` / `f1_macro_after` 越高越好。
- `accuracy_drop` 是相对 shared_base 的下降量；负数表示 unlearning 后测试准确率反而高于 base artifact 的测试口径。
- `overall_mia_auc` 越接近 0.5 越好，表示 membership inference attack 越接近随机猜测。
- `privacy_score` 越高越好，是当前评估脚本给出的隐私友好分数。
- `online_wall_clock_seconds` 是当前 JSON 中记录的在线耗时；不同方法的预处理/缓存策略不同，所以论文中应把效率指标和 utility/privacy 分开解释。

### 3.2 数据集适用性与风险

| 数据集 | 当前状态 | 适合点 | 主要问题/论文写法 |
| --- | --- | --- | --- |
| PubMed | 当前主实验 | 医学 citation graph；3 类节点分类；shared_base train_acc 约 0.875-0.879，明显高于 39.9% 多数类。 | 不是 KG，类别少；适合作为医学 citation graph 主实验/稳定性基线。 |
| PrimeKG-NoSource | 当前主实验 | 医学 KG 子图；disease 与 gene/protein 二分类；显式移除了 entity-type one-hot 和 node-source one-hot。 | 二分类且多数类约 61.9%；结构特征较少，任务是 KG 子图适配版，不是完整多关系 KG benchmark。 |
| PrimeKG disease-gene-small | 归档/不建议主表 | shared_base train_acc 可达约 0.985。 | `node_source_one_hot` 与 label 高度相关，存在捷径/公平性质疑；应使用 NoSource 版本替代。 |
| PPI single-label variants | 探索过，不纳入主表 | 医学/生物蛋白互作数据，原始任务本身有价值。 | 原生 PPI 是 multi-label；single-label projection 要么塌缩为 92% 多数类，要么 balanced 后 GCN/GraphSAGE utility 不成立。未来若支持 multi-label，可重新接入。 |
| Cora/CiteSeer | 历史/非医学 | 有 shared_base 与旧实验结果，可用于工程 sanity。 | 非医学数据集；不适合支撑当前医学图遗忘论文主叙事。 |
| ogbn-arxiv | 已删除/不纳入 | 大图 citation benchmark。 | 非医学；HASI full inpainting/DAR 在该图上图处理代价过高，且当前已从代码和 artifacts 清理。 |

## 4. 主结果总览：HASI-tuned vs baselines

本节只汇总当前主线 tuned HASI 与 baselines。PubMed node 使用 `inpainting_repair_fix` 版本，PubMed edge 使用 `hasi_edge_latest_tuned`，PubMed feature 使用 `hasi_tuned`；PrimeKG-NoSource 使用对应 tuned 结果。

结果覆盖：当前扫描到有效主结果 JSON 共 `156` 个；跳过的 JSON 共 `72` 个，主要是 GraphEraser artifact metadata/partition 文件，不是实验结果。

公平性复核：对 36 个 dataset / type / seed / ratio 主结果组逐组检查后，每组内的 `forget_set.path`、`base_artifact.path`、forget seed 和目标数量均一致，未发现 HASI 与 baselines 的主表错配。

## 7. 论文写作建议

1. 主表建议只放 `PubMed` 和 `PrimeKG-NoSource`，按 dataset / unlearning type / ratio 汇总三 seed 均值，再把 per-seed 细表放附录或补充材料。
2. 数据集方法段要明确：shared_base 训练只用 `train_subgraph`，forget set 与 base artifact 绑定，node forget 不从 val/test 抽。
3. PPI 不要写成失败数据集，而写成“原生 multi-label benchmark，当前框架 single-label 分类头不适配；完整 multi-label unlearning 支持留作未来工作”。
4. PrimeKG-NoSource 要强调去掉 source feature 是为了避免 label-correlated shortcut；这比带 source 的高精度结果更公平。
5. HASI 的强项目前更偏 utility-preserving 和结构/表示修复；效率上 full inpainting 和 DAR 很重，应如实讨论，而不是只展示 utility。
